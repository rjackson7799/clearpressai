# ClearPress AI 操作ガイド（社内スタッフ向け）

> 本ガイドは、ClearPress AI を使う **社内スタッフ** 向けの操作手順書です。クライアント登録からブランドボイス抽出、原稿生成、薬機法コンプライアンス確認、監査レポートの署名、メール配信、フィードバック確認までの一連の流れを、実際の画面に沿って説明します。
>
> 新しく担当になった方は、上から順に読み進めれば、ログインから最初の配信まで一通り実施できます。特定の操作だけ確認したい場合は、下の目次から該当セクションへ移動してください。
>
> **クライアント（配信先）向けのフィードバック画面（マジックリンク）については本ガイドの対象外です。** 受信者は届いたメールのリンクを開いて案を選ぶだけで、専用の説明は不要です。

## 1. はじめに

### ClearPress AI とは

ClearPress AI は、製薬 PR を扱う社内チームが、**特定のクライアントのブランドボイス**でプレスリリース等のコンテンツを生成し、**薬機法・適正広告基準・PMDA ガイドライン**に沿ったコンプライアンス確認を行い、監査記録を残したうえでクライアントにメール配信するための社内ツールです。

### 全体の流れ

作業は次の順序で進みます。各ステップは前のステップの完了を前提としています。

```
クライアント作成
   ↓
ブランドボイスの抽出（過去原稿から）
   ↓
プロジェクト（ブリーフ）作成 → 3案を自動生成
   ↓
バリアントのレビューと承認
   ↓
コンプライアンスチェック（指摘の解決）
   ↓
監査レポートの作成と署名（確定）
   ↓
配信の作成（メール送信・予約）
   ↓
クライアントからのフィードバック確認
```

### 重要な用語

| 用語 | 意味 |
| --- | --- |
| **ブランドボイス** | クライアント固有の文体・語彙・トーン。過去原稿から AI が抽出したプロファイル。 |
| **バリアント（案）** | 1つのブリーフから生成される3つの原稿候補。画面上では「案1」「案2」「案3」と表示されます。 |
| **コンプライアンスチェック** | 生成した原稿に対する薬機法等の確認。決定的チェック（自動ルール）と LLM レビューの2段階。 |
| **指摘（Finding）** | コンプライアンスチェックで見つかった問題。重大度は「阻止 / 警告 / 注意」の3段階。 |
| **監査レポート** | 承認した案・コンプライアンス結果・承認者などを固定した記録。署名すると確定します。 |
| **署名 / 確定** | 監査レポートに電子署名を付けて内容を固定する操作。確定後は編集できません（改訂が必要）。 |
| **配信** | 承認済みの案をクライアントにメール送信すること。PDF / Word を添付できます。 |
| **フィードバック** | 配信先クライアントがマジックリンクから返す評価。選んだ案・コメント等。 |

> **なぜ手順が固定されているのか**
> 監査レポートの署名や配信には、規制対応のための「前提条件（ゲート）」が設定されています。たとえば「コンプライアンスの阻止指摘が未解決のままでは署名できない」など。これらは意図的な安全装置で、順番を飛ばすとエラーになります。各セクションの「注意」ブロックで、どの条件が必要かを説明します。

## 2. ログインと基本操作

### ログイン（`/login`）

1. ログイン画面で **メールアドレス** と **パスワード** を入力します。
2. 「ログイン」ボタンを押します。
   - メールとパスワードの両方を入力するまでボタンは押せません。
3. パスワードを使わずログインしたい場合は、メールアドレスだけ入力して「マジックリンクで送信」を押すと、ログイン用リンクがメールで届きます。送信に成功すると「マジックリンクで送信 ✓」と表示されます。

> **注意：パスワードは12文字以上**
> パスワードは12文字以上である必要があります。画面上には明示されていませんが、短いと保存・ログインに失敗します。

### パスワードの再設定

- ログイン画面下部の「パスワードをお忘れの方」を押すと再設定画面（`/forgot-password`）に移動します。
- メールアドレスを入力して「パスワード再設定」を押すと、再設定用リンクがメールで届きます（「パスワード再設定 ✓」と表示）。
- 届いたリンクを開くと新しいパスワードの入力画面（`/reset-password`）が開きます。**12文字以上** の新しいパスワードを入力して「保存」を押すと、ダッシュボードに移動します。

### 画面の基本構成

ログイン後の画面は、左側の **サイドバー**、上部の **ヘッダー**、中央の作業エリアで構成されます。

- **サイドバー**：「ダッシュボード」「クライアント」「プロジェクト」「設定」の4項目。
- **ヘッダー**：右側に **言語切り替え** と「ログアウト」ボタン。
- 変更中の言語は、ヘッダーの言語切り替えで「日本語 / English」を選ぶと切り替わります。設定はアカウントに保存されます。

> **注意：言語切り替えの場所**
> 言語切り替えは **ログイン後のヘッダー** と **設定画面** の2か所にのみあります。ログイン・パスワード再設定などのログイン前の画面には表示されません。

> **補足：クライアント／プロジェクトへの入り口**
> サイドバーのトップ項目は4つだけです。原稿レビューやコンプライアンス、監査、配信などの画面は、「クライアント」や「プロジェクト」の一覧から個別のクライアント／プロジェクトを開いて辿っていきます。

## 3. ダッシュボードと「はじめに」チェックリスト

ログイン直後に表示される「ダッシュボード」には、初めて使う方向けの **「はじめに」チェックリスト** があります。

### 5つのステップ

「はじめに」カードには次の5ステップが並びます。完了したステップには緑のチェックが付き、取り消し線が表示されます。未完了のうち先頭のものにヒントと「開始 →」ボタンが表示されます。

1. **クライアントを作成** — 最初のクライアントを登録します。
2. **ブランドボイス資料を追加** — クライアントの過去の文章をアップロードし、ブランドボイスを抽出します。
3. **最初のプロジェクトを作成** — ブリーフを入力してコンテンツの生成を開始します。
4. **バリアントをレビューして承認** — 生成されたバリアントを比較し、1つを承認します。
5. **クライアントに配信** — 承認済みのコンテンツをメールで送付します。

> **注意：ステップ4・5が「完了」になる条件**
> - ステップ4は、いずれかのプロジェクトで少なくとも1案を **承認済み** にすると完了になります（レビュー画面を開いただけでは完了しません）。
> - ステップ5は、いずれかのプロジェクトの状態が **「送付済」または「フィードバック受領」** になると完了します。

### カードの非表示

チェックリストが不要になったら「非表示にする」を押すと消えます。また、5ステップすべてが完了すると自動的に非表示になります。

## 4. クライアントの作成と管理

### クライアントの新規作成（`/clients/new`）

1. サイドバーの「クライアント」を開き、「新規作成」を押します（または「はじめに」チェックリストの「開始 →」）。
2. 「新規クライアント」画面で以下を入力します。
   - **クライアント名**（必須）
   - **英語名**
   - **担当者名**
   - **担当者メール**（メール形式のチェックあり）
   - **メモ**
3. 「作成」を押すと保存され、そのクライアントの詳細画面に移動します。

> **補足：業界の選択欄はありません**
> 業界は「製薬」に固定されているため、入力欄はありません。

### クライアント詳細画面（`/clients/:id`）

詳細画面には **4つのタブ** があります。

| タブ | 内容 |
| --- | --- |
| **詳細** | 基本情報の表示。「編集」ボタンで編集フォームに切り替わります。 |
| **ブランドボイス** | 過去原稿のアップロードとブランドボイスの抽出（→ [第5章](#5-ブランドボイスの抽出)）。 |
| **ガイドライン** | ブランドに関する内部メモ・ガイドラインの管理。 |
| **フィードバック** | クライアントから返ってきたフィードバックの確認（→ [第11章](#11-フィードバックの確認)）。 |

- 右上の「編集」で基本情報を修正できます。
- 「削除」を押すと確認ダイアログ（「このクライアントを削除しますか？」）が表示されます。**削除するとブランドボイス・ガイドラインも一緒に削除され、取り消せません。**

## 5. ブランドボイスの抽出

クライアント詳細画面の **「ブランドボイス」タブ** で行います。過去の原稿を複数アップロードし、そこから AI が文体プロファイルを抽出します。上から「準備状況 → アップロード → サンプル一覧 → プロファイル編集」の順に並んでいます。

### 手順

1. **資料をアップロードする**
   - アップロード欄に **PDF / DOCX / TXT** ファイルをドラッグ＆ドロップ、またはクリックして選択します。
   - 対応ファイル：**5〜20件、各10MB以下**。
   - アップロードした各ファイルには「待機中 / 抽出中 / アップロード中 / 完了」などの状態が表示されます。
2. **準備状況を確認する**
   - 上部の準備状況バーに、分析に使えるサンプル数が表示されます。
   - 十分になると緑色で「分析に十分です (件数/5+)」と表示されます。
   - 不足している場合は「あと◯件のサンプルが必要です」と表示されます。
3. **抽出を実行する**
   - 準備が整うと「抽出を実行」ボタンが押せるようになります。
   - 実行中は「抽出中…」と表示されます。完了すると「ブランドボイスを抽出しました」と通知されます。
4. **プロファイルを確認・編集する**（下記参照）。

> **注意：「抽出を実行」ボタンが押せない場合**
> 抽出には **「使用可能なサンプル」が5件以上** 必要です。ここでいう「使用可能」とは、**抽出されたテキストが500文字以上** のサンプルを指します。
> - スキャンした PDF など、文字が読み取れないファイルは「テキストが少なすぎます（スキャンPDFの可能性）」と警告され、カウントされません。
> - サンプル一覧の文字数バッジが **赤色（500文字未満）** のファイルは分析対象外です。テキスト量の多いファイルに差し替えてください。
> - 一部が除外された場合、「テキストが少ないサンプルは含まれません」と表示されます。

> **注意：再抽出すると手動編集が上書きされます**
> プロファイルを手動で編集したあとに再度「抽出を実行」しようとすると、「手動編集が破棄されます」という確認ダイアログが表示されます。続行すると、手で直した内容は AI の出力で上書きされます。

### プロファイル編集フォーム

抽出後、以下の項目を編集できます（タグ形式の項目は Enter またはカンマで追加）。

- **トーンキーワード**（最大7個）
- **文体的特徴**（自由記述）
- **好まれる語彙**（最大15個）
- **避けるべき表現**（最大10個）
- **シグネチャーフレーズ**（最大6個）
- **標準的な分量**（コンテンツ種別ごとの目安。「種類を追加」で行を増やせます）

編集後は「保存」を押します（「プロファイルを保存しました」と通知）。「キャンセル」で編集内容を破棄できます。

### ガイドライン（「ガイドライン」タブ）

- 「内部メモを追加」の欄にテキストを入力し「追加」を押すと、内部メモとしてガイドラインに登録されます。
- 各ガイドラインには出所を示すバッジが付きます：**抽出 / 内部メモ / クライアントフィードバック / リーガルレビュー**。
- 不要になったガイドラインは「アーカイブ」で片付けられます（確認あり）。

## 6. プロジェクト（ブリーフ）の作成

新しい原稿を作るには「プロジェクト」を作成し、ブリーフ（指示書）を入力します。作成すると自動的に3案の生成が始まります。

### 手順（`/projects/new`）

1. サイドバーの「プロジェクト」→「新規プロジェクト」を開きます。
2. 「新規プロジェクト」画面で以下を入力します。

| 項目 | 説明 |
| --- | --- |
| **クライアント**（必須） | 対象クライアントを選択します。 |
| **プロジェクト名**（必須） | 案件名。 |
| **コンテンツ種別** | プレスリリース / ブログ / SNS投稿 / 社内メモ / FAQ / エグゼクティブステートメント。 |
| **サブタイプ** | **「プレスリリース」を選んだときのみ表示**。自動判定 / 完全臨床発表 / パートナー謝辞 / CSR/イベント / ビジネスニュース。 |
| **緊急度** | 通常 / 優先 / 緊急 / 危機対応。 |
| **締切** | 任意（日付）。 |
| **バリエーション軸** | 3案の作り分けの軸：トーン / 構成 / 長さ。 |
| **言語** | 日本語 / English。 |
| **ブリーフ本文** | 原稿の指示（**50文字以上必須**）。 |
| **キーメッセージ** | 伝えたい要点（タグ形式）。 |
| **引用** | 氏名・役職・発言のセット。「引用を追加」で複数登録可（各項目とも入力必須）。 |
| **データ** | 数値・データポイント（タグ形式）。 |
| **制約・注意事項** | 任意。 |

3. 「3案を生成」を押すと、プロジェクトが作成され（「プロジェクトを作成しました」）、そのままレビュー画面に移動して生成が始まります。

> **補足：ブリーフは具体的なほど良い**
> ブリーフ本文は最低50文字ですが、**200文字以上** になると入力欄に緑のチェックが付きます（内容が充実したブリーフの目安）。背景・目的・伝えたい点を具体的に書くほど、生成される原稿の質が上がります。

> **注意：この種別の標準分量がプロファイルにない場合**
> 選んだクライアントのブランドボイスに、選択したコンテンツ種別の「標準的な分量」が登録されていない場合、「ボイスプロファイルにこのコンテンツ種別の標準分量がありません」という案内が表示され、代替の分量（プレスリリース等）が使われます。問題なく生成は続行できます。

> **補足：サブタイプ欄が消える理由**
> サブタイプは「プレスリリース」専用の項目です。他の種別を選ぶと自動的に「自動判定」になり、欄そのものが非表示になります。

### プロジェクト一覧（`/projects`）

一覧には各プロジェクトの状態バッジ（**下書き / レビュー中 / 送付済 / フィードバック受領 / 完了**）が表示されます。状態が「レビュー中」または「送付済」の行には「配信 →」のショートカットが出ます。

## 7. バリアントのレビューと承認

プロジェクト作成後に開くレビュー画面（`/projects/:id/review`）で、3つの案を見比べて編集・承認します。

### 自動生成

- 案がまだ無い状態でこの画面を開くと、**自動的に3案の生成 → コンプライアンスチェック** が始まります。
- 生成中は進捗バーと「3案を生成中...」が表示されます（3つの枠がプレースホルダー表示）。

### 3案の画面構成

各案は縦1列（合計3列）で表示され、それぞれに次の要素があります。

- ヘッダー：「案1」などのバッジ、案のラベル、承認済みなら「承認済」バッジ。
- **「承認」ボタン**（承認済みの場合は「承認を取り消す」）。
- **「再生成」ボタン**（その案だけ作り直す）。
- 本文エディタ（後述）。
- 指摘バー：コンプライアンスの指摘がある場合に「指摘を見る」と件数バッジが表示されます（→ [第8章](#8-コンプライアンスチェック)）。
- 文字数と読了時間の目安。

### 本文の編集と自動保存

- 本文はその場で編集できます。
- 入力を止めてから **約1.5秒後に自動保存** されます。状態表示は「保存中… → 未保存の変更があります → 保存済」と変化します。
- 手動で保存したい場合は「保存」ボタンを押します（未保存の変更があるときのみ押せます）。

> **注意：「承認」ボタンが押せない（グレーアウト）とき**
> 本文に **未保存の変更（編集途中）** があると、「承認」「再生成」ボタンは押せません。これは編集途中の内容を誤って確定しないための仕様です。
> **対処：まず本文を保存してください。**「保存済」になればボタンが有効になります。

### 再生成

「再生成」を押すと確認ダイアログ（「案◯を再生成しますか？」）が出ます。**再生成すると現在の本文とコンプライアンス検査結果は削除されます。**

### 承認

「承認」を押すとその案が承認済みになります（「案を承認しました」）。監査レポートの作成には最低1案の承認が必要です。

> **注意：編集ロック中の表示**
> このプロジェクトの監査レポートがすでに **確定（署名済み）** されている場合、レビュー画面上部に「編集ロック中」と表示され、本文編集・承認・再生成ができなくなります。修正するには監査画面で「改訂を要求」してロックを解除してください（→ [第9章](#9-監査レポートと署名)）。

## 8. コンプライアンスチェック

原稿に対する薬機法・適正広告基準・PMDA ガイドラインの確認結果を確認し、指摘を解決します。レビュー画面で案の「指摘を見る」を押すと、右側にコンプライアンスパネルが開きます。

### パネルの構成

- 上部に「案1 / 案2 / 案3」のタブ。各タブには未解決の **阻止（重大）** 件数が赤バッジで表示されます。
- 指摘は重大度ごとにグループ化されます（順に **阻止 → 警告 → 注意**）。

### 指摘カードの見方と対処

各指摘カードには、重大度、該当箇所（`¶` 段落番号つき）、指摘対象の本文、説明、根拠となる規定が表示されます。修正案がある場合は「修正案」ボックスとコピーボタンが出ます。対処ボタンは次の3つです。

| ボタン | 動作 |
| --- | --- |
| **修正を適用** | 修正案の内容で本文の該当箇所を自動置換します（修正案があるときのみ表示）。適用後は状態が「修正済」になります。 |
| **確認済み** | 修正はしないが、内容を確認・許容したものとして扱います。状態が「確認済」になります。 |
| **未対応に戻す** | 「修正済」「確認済」を取り消し、未対応に戻します。 |

> **注意：「修正を適用」で「本文に該当箇所が見つかりません」と出るとき**
> 本文をすでに手で編集していて、指摘対象の文が本文中に見つからない場合、置換は行われず「本文に該当箇所が見つかりません」と表示されます。この場合は本文を目視で確認し、手動で修正してください。

### 再チェック（Stale）

本文を編集すると、その案のコンプライアンス結果が古くなります。この状態では「再チェックが必要」バナー（および案の列に「要再チェック」フラグ）が表示されます。「再チェック」ボタンで最新の本文に対して再実行してください。

> **注意：署名の前提になります**
> 監査レポートの署名（確定）には、次の2つが必須です。
> 1. **阻止（重大）の指摘がすべて解決済み**（修正または確認済み）であること。
> 2. **本文を編集したあとに再チェックが実行されている**こと。
>
> これらが満たされていないと、署名時にエラーになります（→ [第9章](#9-監査レポートと署名) の署名エラー一覧）。

## 9. 監査レポートと署名

承認した案・コンプライアンス結果・承認者などを1つの記録にまとめ、電子署名で確定します。監査画面（`/projects/:id/audit`）で行います。レビュー画面右上の「監査レポート」から移動できます。

### 手順の流れ

1. **監査レポートを作成する**
   - レポートが未作成の場合、「このプロジェクトには監査レポートがまだありません」と表示されます。
   - 「監査レポートを作成」を押すと下書きのレポートが作られます（**最低1案の承認が必要**）。
2. **内容を確認する**
   - 下書き状態では「プレビュー: 確定前のライブビュー」と表示され、現時点で承認済みの案が表示されます。
3. **署名して確定する**
   - 「署名して確定」を押すと確認ダイアログが開きます。「署名して確定」で確定します（「署名を完了しました」）。
   - 確定後、レポートの状態は「下書き」→「確定済」に変わり、バージョン（例：V1.0）が付きます。
4. **必要に応じて検証・改訂・印刷する**（下記）。

> **注意：署名できないときのエラーと対処**
> 署名時に前提条件（ゲート）を満たしていないと、「署名を完了できません」という赤いエラーが表示されます。主なものは次のとおりです。

| 表示されるメッセージ | 意味 / 対処 |
| --- | --- |
| 承認された案が1件もありません。署名前に少なくとも1案を承認してください。 | レビュー画面で最低1案を承認してください。 |
| 未解決の阻止指摘事項があります。修正または承認のうえ、再チェックしてください。 | 阻止（重大）の指摘を「修正を適用」または「確認済み」にしてください。 |
| 本文編集後にコンプライアンス再チェックが行われていません。再チェックしてから署名してください。 | コンプライアンスパネルで「再チェック」を実行してください。 |
| 承認済みの案にコンプライアンスチェック履歴がありません。再チェックを実行してください。 | その案のコンプライアンスチェックを一度も実行していません。再チェックしてください。 |
| スナップショットがサーバーの状態と一致しません。ページを再読み込みして再試行してください。 | 他の変更と競合しています。ページを再読み込みしてから署名し直してください。 |

### 署名の検証

確定後、「署名」セクションの各署名に付く「署名を検証」ボタンで正当性を確認できます。成功すると「検証成功」（緑）、不一致の場合は「不一致」（赤）と表示されます。

### 改訂（バージョン更新）

確定後に修正が必要になった場合は「改訂を要求」を押します。

1. ダイアログで「改訂理由」を入力します（例：「クライアントから誇大表現の指摘を受けたため再修正」）。
2. 「改訂版を作成」を押すと、現在のレポートを複製した新しい下書きが作られます。確定済みのレポートはアーカイブとして残ります。
3. 新しい下書きに切り替わると、レビュー画面の編集ロックが解除され、再び編集・承認できるようになります。

> **注意：改訂できないときのエラー**
> - 「確定済のレポートのみ改訂できます」…下書きのレポートは改訂できません。
> - 「このバージョンは既に改訂済です。最新版から改訂を開始してください。」…古いバージョンからは改訂できません。最新版を開いてください。
> - 「改訂作業中のドラフトが既に存在します。」…すでに改訂中の下書きがあります。「改訂中の下書きを開く」で続きから作業してください。

### 印刷

確定済み（または改訂済み）のレポートは「印刷」ボタンから印刷用ページ（`/print/audit-report/:id`）を新しいタブで開けます。

### 配信へ進む

確定済みになると「配信を作成」ボタンが表示され、配信画面へ進めます（→ [第10章](#10-配信の作成)）。

## 10. 配信の作成

承認済みの案をクライアントにメール送信します。配信画面（`/projects/:id/deliver`）で作成します。監査画面の「配信を作成」から移動できます。

> **前提：監査レポートの署名が必須**
> 配信するには、そのプロジェクトの監査レポートが **確定（署名済み）** されている必要があります。未署名の場合は先に署名してください（→ [第9章](#9-監査レポートと署名)）。

### 画面の各セクション

配信画面は上から順に次のセクションで構成されます。

1. **配信する案** — 承認済みの案から送るものを選びます（**最大3件**）。1つ選ぶと、別のコンテンツに属する案は選べなくなります（同じコンテンツの案のみまとめて配信）。各案の「推奨」で、おすすめの案を1つ指定できます。
2. **宛先** — メールアドレス・氏名・CC を入力します。
3. **送信者** — 差出人情報（固定）と Reply-To が表示されます。「社内BCCは自動で適用されます」との注記のとおり、社内向け BCC はサーバー側で自動付与されます。
4. **件名** — 「プロジェクト名 — 種別」の形で自動入力されます。編集可能です。
5. **本文** — 日本語の挨拶文と案の比較サマリーが自動入力されます。送信時にサーバーが末尾に **フィードバック用フッター** を自動追加します（プレビュー表示あり）。
6. **添付形式** — **PDF / Word / 両方**（初期値は「両方」）。
7. **送信タイミング** — 「今すぐ送信」または「予約送信」。予約する場合は **日本時間（JST）** で日時を指定します。
8. **送信前チェックリスト** — 下記の4項目。

### 送信前チェックリスト（4項目）

送信ボタンは、次の4項目がすべて満たされるまで押せません。

1. **監査レポートが署名済み** — 未署名なら監査画面で署名してください。
2. **選択した案がすべて承認済みで、承認後に編集されていない** — 案が未選択の場合や、承認後に編集された案がある場合は満たされません（再承認するか選択を外す）。
3. **宛先メールアドレスの形式が正しい** — 有効なメールアドレスを入力してください。
4. **宛先と本文を自分の目で確認した**（手動チェック） — 送信前の最終確認として、自分でチェックを入れる必要があります。

すべて満たすと「今すぐ送信」または「予約送信」ボタンが有効になります。

> **注意：営業時間外・祝日の予約**
> 予約送信の指定時刻が営業時間外（JST 09:00–18:00 の外）や日本の祝日にあたる場合、注意事項が表示されます。送信時に「送信タイミングの確認」ダイアログが出るので、内容を確認して「このまま続行」を押します。確認した内容は監査トレイルに記録されます。

> **注意：送信に失敗したときの主なメッセージ**
> - 「監査レポートが署名済みではありません。監査ページから署名してください。」
> - 「監査確定後に編集された案があります。改訂版の監査レポートを作成してください。」…署名後に案を編集した場合。監査を改訂して署名し直してください。
> - 「未承認の案が含まれています。レビュー画面で承認してください。」
> - 「予約送信時刻が過去になっています。未来の時刻を指定してください。」
> - 「サーバー設定が不完全です。管理者にお問い合わせください。」…差出人設定が未整備の場合。管理者に連絡してください。

送信に成功すると「配信を送信しました」（予約時は「◯◯ に配信を予約しました」）と通知され、配信履歴に移動します。

### 配信履歴（`/projects/:id/deliveries`）

- 作成日時・宛先・案数・添付形式（PDF / Word / PDF + Word）・状態（**下書き / 予約済 / 送信済 / 失敗**）・送信日時が一覧表示されます。
- 「新規配信」から追加の配信を作成できます。

## 11. フィードバックの確認

配信を受け取ったクライアントは、メール内のマジックリンクから案を選び、コメントを返します。**社内スタッフは、その結果をクライアント詳細画面の「フィードバック」タブで確認します**（クライアント側の入力画面は本ガイドの対象外です）。

### 見方

「フィードバック」タブには、返ってきたフィードバックがカードで表示されます。各カードには次が含まれます。

- 配信の件名・宛先名・送信日時、および元の配信への「配信を見る」リンク。
- クライアントの選択結果：**「再検討が必要」**、または **「案◯ · （案のラベル）」**（選ばれた案）。
- **良かった点 / 改善できる点**（チップ）と自由記述の **コメント**。
- そのフィードバックから自動生成された **ガイドライン**（「生成されたガイドライン (件数)」）。生成されたガイドラインは「ガイドライン」タブにも反映され、出所バッジは「クライアントフィードバック」になります。

### ガイドライン生成が失敗した場合

- フィードバックから自動でガイドラインを作る処理が失敗すると、「ガイドライン抽出失敗」バッジとエラー内容が表示されます。
- 「再試行」ボタンで再実行できます（成功すると「ガイドラインを再生成しました」）。

## 13. 困ったときは（FAQ・トラブルシューティング）

「操作したいのにボタンが押せない／エラーが出る」ときの主な原因と対処をまとめます。

### 「抽出を実行」ボタンが押せない
ブランドボイスの抽出には、**500文字以上のサンプルが5件以上** 必要です。文字数バッジが赤いファイル（500文字未満、スキャンPDFなど）はカウントされません。テキスト量の多いファイルに差し替えてください。（→ [第5章](#5-ブランドボイスの抽出)）

### 「承認」「再生成」ボタンがグレーアウトして押せない
本文に **未保存の変更（編集途中）** があるためです。まず「保存」して「保存済」にしてください。監査レポートが確定済みで「編集ロック中」の場合もロックされます（改訂で解除）。（→ [第7章](#7-バリアントのレビューと承認)）

### サブタイプの欄が見当たらない
サブタイプは **「プレスリリース」を選んだときだけ** 表示されます。他の種別では非表示です。（→ [第6章](#6-プロジェクトブリーフの作成)）

### 「署名して確定」でエラーになる
主な原因は、①承認済みの案がない、②阻止（重大）の指摘が未解決、③本文編集後に再チェックしていない、④コンプライアンス未実行、⑤サーバー状態との不整合、のいずれかです。表示されるメッセージに従って対処してください。（→ [第9章](#9-監査レポートと署名) の署名エラー一覧）

### 「改訂を要求」が表示されない／押せない
改訂は **確定済みの最新バージョン** に対してのみ可能です。すでに改訂済みの古いバージョンや、改訂中の下書きがある場合は表示条件が変わります。最新版を開き直してください。（→ [第9章](#9-監査レポートと署名)）

### 「送信」ボタンが押せない
**送信前チェックリストの4項目** がすべて満たされていない可能性が高いです。特に4番目の「宛先と本文を自分の目で確認した」は手動でチェックを入れる必要があります。（→ [第10章](#10-配信の作成)）

### 配信で「監査確定後に編集された案があります」と出る
監査レポートを署名したあとに案の本文を編集すると、配信がブロックされます。監査画面で「改訂を要求」して改訂版を作成し、署名し直してください。（→ [第9章](#9-監査レポートと署名) / [第10章](#10-配信の作成)）

### 配信で「サーバー設定が不完全です」と出る
差出人などのサーバー側設定が未整備です。スタッフでは解決できないため、管理者にお問い合わせください。

### 言語切り替えが見つからない
言語切り替えは **ログイン後のヘッダー右側** と **設定画面** にあります。ログイン前の画面にはありません。（→ [第2章](#2-ログインと基本操作)）

### 「修正を適用」で「本文に該当箇所が見つかりません」と出る
本文をすでに手で編集していて、指摘対象の文が本文中に残っていない場合に出ます。本文を目視で確認し、手動で修正してください。（→ [第8章](#8-コンプライアンスチェック)）
